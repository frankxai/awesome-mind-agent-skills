# Contributing to Awesome Mind Agent Skills

Thanks for helping build the map of agentic systems and skills for understanding, structuring, and extending the human mind.

## What qualifies

An entry earns a place if it helps someone **model** their own mind — its attention, memory, emotion, belief, or any of the [12 constructs](https://github.com/frankxai/mind-intelligence-systems/tree/main/models/human-mind) — and do something with that model. A repo, a portable `SKILL.md`, a research runtime, or a related ecosystem that genuinely composes with the swarm all qualify. A link with no working artifact behind it does not.

## Guidelines

- **One addition per pull request.** Easier to review, easier to discuss.
- **Keep links live.** No dead links, no tracker redirects. Link the official repo, skill file, or model file — never an invented deep link.
- **Match the format.** Use the existing table or bullet style of the section you're adding to.
- **Say why it belongs.** One short clause on what makes the entry worth a reader's time. "It exists" is not a reason.
- **Status is honest.** `live` means it ships today. `planned` means it's chartered but not built — say so. Don't dress a stub up as a system.
- **Verify claims.** No fabricated stats, no overstated maturity. If you can't source it, don't claim it.

### Format of an entry

A repo or skill entry is a single line:

```
- [name](https://github.com/owner/repo) — one clause: what it is + why it earns the slot.
```

For the ecosystem tables, fill all columns (Repo · Role · Status). Keep the role to one sentence.

## Voice rules

- **Direct, technical, warm.** Write like you're handing a peer a map, not pitching them.
- **No AI-slop.** No "delve", "dive into", "unlock", "harness the power", "seamless", "elevate". Name the mechanism instead.
- **Never clinical or diagnostic.** These systems model a mind; they never diagnose one. Nothing with a `-therapy`, `-diagnosis`, or `-disorder` framing gets merged. If something blurs observation into diagnosis, it's out.
- **No self-promotion without substance.** Linking your own project is fine — if it works and earns the slot on the same bar as everything else.

## What doesn't get merged

- Clinical, diagnostic, or therapeutic tooling dressed as a "mind" system.
- Dead, closed, or vaporware projects (mark existing ones if they lapse).
- Entries with no working artifact, or with a one-line reason that's pure marketing.
- `planned` work listed as `live`.

Open a PR. Be honest. Keep the map true to what actually ships. That's the whole bar.
